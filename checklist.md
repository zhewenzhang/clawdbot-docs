# 执行检查清单

**每次收到任务必须做：**

## 执行前 - 强制检查

### □ 1. 任务识别
- 任务类型：______（分析/修复/规划/创建/审查/确认/简单）
- 复杂度评分（1-10）：__

### □ 2. 方法匹配
- [ ] 直接执行（1-3分）
- [ ] 深度思考（4-6分）
- [ ] 深度思考 + Skill（7-10分）
- 需要调用哪个 Skill：______

### □ 3. 用户确认
向用户确认：
```
收到任务，让我先确认：
【任务理解】您的需求是：______
【复杂度评估】预计需要：[深度思考 + Skill / 深度思考 / 直接执行]
【下一步】是否正确？需要调整吗？
```
**只有用户确认后才能开始执行！**

## 执行中 - 可选检查

- [ ] 遇到困难时 → 停下来，用深度思考分析
- [ ] 偏离目标时 → 回到检查清单，重新确认
- [ ] 完成关键步骤 → 简要汇报进展

## 执行后 - 必须检查

- [ ] 结果是否满足用户需求？
- [ ] 是否需要验证？
- [ ] 是否还有其他需求？

---

## 复杂度评分参考

满足任一条件+2分：
- 需要多步骤推理？
- 需要分析多个因素？
- 结果影响重大？
- 涉及代码生成？
- 需要系统性方法？

总分：
- 1-3分 → 简单任务，直接执行
- 4-6分 → 中等任务，深度思考
- 7-10分 → 复杂任务，深度思考 + Skill

---

## 内部默念（强制）

```
1. 我判断任务类型了吗？
2. 我选择正确方法了吗？
3. 我向用户确认了吗？

没有完成 = 不能开始执行
```
