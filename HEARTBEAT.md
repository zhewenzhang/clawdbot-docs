# HEARTBEAT.md - 心跳检查配置

## 检查项目

### 每日检查
- **邮箱**: 检查 davezhangus@gmail.com 是否有重要邮件
- **日历**: 查看今日和明日日程，标记 upcoming events
- **天气**: 根据老板出行需求决定是否检查

### 每周检查
- **网站监控**: 半导体相关网站（待添加具体列表）
- **半导体数据库**: 每周一 10:00 扫描新公司

### 每月检查
- **半导体数据库**: 每月1号 9:00 更新 Roadmap 数据

## 主动联系策略

### 联系频率
- 每天最多 3 次主动联系（上午、下午、晚上）
- 每次联系内容精炼，不冗长

### 内容优先级
1. **必须通知**: 重要邮件到达、日历事件 (<2h)
2. **常规更新**: 任务完成、报告生成、分析结果
3. **有趣发现**: 行业新闻、技术趋势

### 深夜时段 (23:00-08:00)
- 留言为主，不过多打扰
- 除非紧急，否则不主动发送大量内容

## 状态追踪
检查状态保存在 `memory/heartbeat-state.json`

```json
{
  "lastChecks": {
    "email": null,
    "calendar": null,
    "weather": null,
    "websites": null
  }
}
```

## 快捷指令
在聊天中可直接使用：
- `/*邮件` - 手动触发邮箱检查
- `/*日历` - 查看今日日程
- `/*今日总结` - 快速回顾今天任务

## 注意事项
- 不要过于频繁检查（间隔 <30 分钟不重复）
- 老板忙碌时减少主动联系
- 数据品质优先，速度次之
