### 2.2 优先级排序

```
P0（必须查阅）:
├── 公司年报（Form 10-K）

└── 官方产品发布公告

├── 行业新闻媒体报道
├── 专利数据库
└── 社交媒体/技术博客
```

## 3. 数据收集流程

### 3.1 信息收集清单

#### 3.1.1 基础信息收集

| 字段 | 数据来源 | 验证方法 | 常见问题 |
|------|----------|----------|----------|
| 公司名称 | 公司官网/年报 | 核对官方英文名 | 中文名可能有多种翻译 |
| 股票代码 | 交易所官网 | 多地上市需分别确认 | 部分公司从多个交易所退市 |
| 成立时间 | 公司官网/维基 | 交叉验证 | 可能有收购导致的继承关系 |
| 总部位置 | 公司官网 | 核对最新信息 | 可能有多个全球总部 |
| 员工人数 | 年报 | 与官网对比 | 不同年份数据可能变化 |

#### 3.1.2 芯片类型分类

| 字段 | 数据来源 | 验证方法 | 注意事项 |
|------|----------|----------|----------|
| 芯片类型 | 产品组合分析 | 查看主要营收来源 | 一家公司可能涉及多种类型 |
| 子类型 | 产品线分析 | 查看具体产品系列 | 更新频繁，需定期维护 |
| 应用领域 | 产品文档 | 查看目标市场 | 一个产品可能覆盖多个领域 |

#### 3.1.3 载板使用分析

| 字段 | 数据来源 | 推断方法 | 可靠性 |
|------|----------|----------|--------|
| ABF/BT载板使用 | 封装技术/产品类型 | 根据芯片性能判断 | 中-高 |
| 封装技术 | 台积电/封装厂文档 | 查看具体产品采用的封装 | 高 |
| 芯片尺寸 | 行业分析/技术文档 | 根据die size估算 | 中 |
| 年用量 | 销售数据/产能 | 公开数据有限，需估算 | 低 |

**载板类型推断规则：**

```
ABF载板适用场景:
├── 高性能CPU（Intel, AMD服务器CPU）
├── 高性能GPU（数据中心GPU，游戏GPU）
├── AI加速器（TPU, Trainium等）
├── 采用CoWoS/2.5D/3D封装的产品
└── 芯片尺寸>400mm²

BT载板适用场景:
├── 手机SoC（Qualcomm, MediaTek, Apple A系列）
├── 显示驱动芯片（Novatek）
├── 存储芯片（DRAM, NAND控制器）
├── 采用Flip Chip传统封装的产品
└── 芯片尺寸<300mm²
```

#### 3.1.4 深度分析字段

| 字段 | 数据来源 | 验证方法 | 备注 |
|------|----------|----------|------|
| 主要客户 | 年报/分析报告 | 查看营收构成 | 大型客户可能占比较高 |
| 晶圆代工 | 年报/技术文档 | 查看工艺节点信息 | 部分为专有信息 |
| 封装测试厂 | 供应链分析 | 第三方估算 | 可能有多个合作伙伴 |
| 技术节点 | 台积电/三星官网 | 确认最新节点 | 量产vs研发 |
| 营收规模 | 年报 | 直接引用 | 需注意币种和单位 |
| 增长预期 | 分析师报告 | 多源对比 | 主观性较强 |

## 4. 数据验证方法

### 4.1 多源交叉验证

每个关键数据点应至少从两个独立来源确认：

```
验证等级:
├── A级：2+官方来源确认（财报+官网）
├── B级：1官方+1行业来源确认
├── C级：仅行业来源估算
└── D级：推测/估算
```

### 4.2 数据质量检查清单

```
数据录入前检查:
[ ] 字段是否完整
[ ] 数据格式是否正确
[ ] 数值是否在合理范围内
[ ] 日期是否有效
[ ] 分类是否准确

数据录入后检查:
[ ] 重复记录检查
[ ] 逻辑一致性检查
[ ] 异常值检测
[ ] 与历史数据对比
[ ] 与同类公司对比
```

### 4.3 数据状态标记

| 状态 | 标记 | 说明 |
|------|------|------|
| 草稿 | Draft | 数据已录入，未验证 |
| 已验证 | Verified | 至少2个来源确认 |
| 待更新 | Pending Update | 发现需要更新的信息 |
| 过期 | Expired | 超过更新周期 |

## 5. 更新频率

### 5.1 按数据类型更新

| 数据类型 | 更新频率 | 触发条件 | 负责人 |
|----------|----------|----------|--------|
| 基础信息 | 季度 | 财报发布后 | 数据管理员 |
| 股票代码 | 实时 | 公司公告 | 数据管理员 |
| 产品信息 | 按需 | 新品发布 | 产品分析师 |
| 营收数据 | 季度 | 财报发布 | 数据录入员 |
| 载板分析 | 半年度 | 行业报告 | 业务分析师 |
| 增长预期 | 半年度 | 分析更新 | 业务分析师 |

### 5.2 更新流程

```
数据更新流程:

Step 1: 监控触发
├── 订阅公司财报发布通知
├── 监控行业新闻
└── 定期行业报告检查

Step 2: 数据收集
├── 下载最新财报
├── 收集产品发布信息
└── 整理行业动态

Step 3: 数据验证
├── 多源交叉验证
├── 与历史数据对比
└── 标记数据状态

Step 4: 数据录入
├── 更新现有记录
├── 添加新记录
└── 记录更新来源

Step 5: 质量审核
├── 数据完整性检查
├── 逻辑一致性检查
└── 确认数据状态
```

### 附录C：常见问题

**Q1: 如何判断一家公司是否使用ABF载板？**

A: 主要通过以下信号判断：
- 产品是否用于高性能计算
- 是否采用CoWoS、InFO等先进封装
- 芯片尺寸是否较大（>400mm²）
- 行业报告是否有明确说明

**Q2: 载板年用量如何估算？**

A: 估算方法：
- 根据营收和产品单价推算出货量
- 参考行业报告的出货数据
- 根据晶圆投片量估算
- 注意区分不同产品线的用量

**Q3: 未上市公司数据如何获取？**

A: 数据来源：
- 行业研究报告
- 供应链访谈
- 专利申请信息
- 新闻媒体报道
- 政府统计数据

**Q4: 芯片类型分类模糊怎么办？**

A: 解决方案：
- 采用多标签分类
- 按主要营收来源分类
- 参考行业标准分类
- 在备注中说明具体情况
