**功能**：代码审查响应
**应用场景**：

**优势**：
- 结构化处理反馈

- 建立Review知识库

---

### Skill 4: test-driven-development
**功能**：测试驱动开发
**应用场景**：
- 新功能开发
- 重构代码
- 建立测试覆盖

**使用方法**：
- RED-GREEN-REFACTOR循环
- 先写测试，再写代码

**优势**：
- 保证代码质量
- 减少回归Bug
- 提升设计质量

**应用场景**：

- 隔离开发环境
- 快速切换上下文

**使用方法**：
```bash

```

**优势**：
- 避免分支混乱
