```
1. 读取 SOUL.md（AI 身份）
2. 读取 personal.md（用户个人框架）⭐
```

## ⚠️ 常见错误与纠正

### 错误1：忘记读取文件
**症状**：直接给出投资建议，没有引用任何框架
**纠正**：
```
❌ 错误：建议买入XX股票
✅ 正确：根据 investment-logic.md 第3条选股标准...
```

### 错误2：框架引用不完整
**症状**：只引用了 investment-logic.md，忽略了 personal.md
**纠正**：
```
❌ 错误：只提到估值符合要求
✅ 正确：估值符合 investment-logic.md，同时符合 personal.md 中用户偏好的[长线]策略
```

### 错误3：决策与框架冲突
**症状**：建议与用户的投资逻辑矛盾
**纠正**：
```
❌ 错误：建议短期炒作
✅ 正确：根据 investment-logic.md，用户偏好长线持有，建议...
```

## 🔧 故障排除

### 问题1：找不到文件
**症状**：`personal.md` 或 `investment-logic.md` 不存在
**解决**：
```bash
ls -la ~/clawd/*.md
# 检查文件是否存在
```

### 问题2：文件内容为空
**症状**：文件存在但全是 `[待补充]`
**解决**：
- 提醒用户补充完整信息
- 先使用通用原则，等待用户更新

### 问题3：框架之间冲突
**症状**：`personal.md` 和 `investment-logic.md` 要求矛盾
**解决**：
- 优先遵循 `investment-logic.md`（投资专业决策）
- 向用户确认哪个优先级更高
