```
1. 【Check】识别任务类型
   - 任务类型：______
   - 推荐Skill：______

2. 【Analyze】分析任务需求
   - 复杂度评分：__/10
   - 优先级：______

3. 【Plan】制定执行计划
   - 步骤：______
   - 预计时间：______

4. 【Do】执行任务
   - 使用推荐Skill
   - 记录进度

5. 【Check】验证结果
   - 验收标准：______
   - 结果：✅/❌

6. 【Adjust】优化调整
   - 优化点：______
   - 知识库更新：______
```

## 三、复杂度判断

| 复杂度 | 时间 | 文件数 | 风险 | 处理方式 |

## 五、错误处理速查

### 错误代码前缀

| 代码 | 类型 | 示例 |
|-----|------|-----|
| F- | 文件操作 | F-001 文件不存在 |
| C- | 命令执行 | C-001 命令超时 |
| P- | 任务规划 | P-001 遗漏步骤 |
| S- | Skills使用 | S-001 Skill不匹配 |
| M- | 沟通错误 | M-001 格式错误 |
| G- | 系统配置 | G-001 规则冲突 |
| R- | 性能问题 | R-001 响应慢 |
| L- | 逻辑错误 | L-001 错误假设 |

### 错误处理流程

```
发现错误
    ↓
分类和定级（P0-P3）
    ↓
┌─────────────────────┐
│ P0/P1 → 立即处理    │
│ P2 → 4小时内处理    │
│ P3 → 24小时内处理   │
└─────────────────────┘
    ↓
根因分析（5-Why）
    ↓
解决方案
    ↓
验证和关闭
    ↓
更新预防措施
```

## 九、常见问题

### Q1: 如何判断使用哪个Skill？
A: 查看任务→Skill匹配表，或使用关键词匹配。

### Q2: 复杂度评分标准是什么？
A: 时间（<5min简单，5-30min中等，>30min复杂）+ 文件数（1-2简单，2-5中等，5+复杂）+ 风险（低/中/高）。

### Q3: 错误发生时怎么办？
A: 触发CAPDCA循环：Check→Analyze→Plan→Do→Check→Adjust。

### Q4: 如何记录循环？
A: 复制模板到memory/capdca-cycles.md，填写各阶段内容。
