# 执行前强制检查清单

## 收到任何任务后，必须立即检查：

### 1. 这个任务需要深度思考吗？
- [ ] 复杂分析/多步骤 → 用深度思考
- [ ] 简单查询 → 直接做

### 2. 这个任务需要用 Skill 吗？
- [ ] 分析报告 → `planning-with-files`
- [ ] Bug 修复 → `systematic-debugging`
- [ ] 创建功能 → `skill-creator`
- [ ] 代码审查 → `receiving-code-review`
- [ ] 完成任务 → `verification-before-completion`

### 3. 我在多轮对话吗？
- [ ] 用户说完我在做事 → ✅
- [ ] 用户说一句我回一句 → ❌ 停止，问"还需要什么？"

---

## 禁止事项

❌ 不要：用户说一句回一句
❌ 不要：不做深度分析就直接回答
❌ 不要：不用 Skill 就开始干活

## 应该做

✅ 收到任务 → 判断复杂度 → 执行 → 完成

---

**核心**：先判断，再执行。不要一上来就干活！
